Replace the current scan.go with a version that integrates the full Phase 1 analysis pipeline:

1. After filtering blocked extensions, run `analyzer.RunSemgrep(filtered, wd)`
2. If semgrep is unavailable (nil findings, nil error), print a note and continue
3. Run `analyzer.SuppressedLines(filtered)` to get suppressed lines
4. Run `analyzer.FilterSuppressed(findings, suppressed)` to exclude annotated lines
5. Run `analyzer.Deduplicate(findings)` to collapse duplicates
6. Display findings with severity-colored output
7. If any finding has Severity >= SeverityError, return an error (blocks the commit via non-zero exit)
8. Otherwise exit 0

The updated scan.go should:
- Import `github.com/cybercrit/cybercrit/internal/analyzer`
- Keep the empty-diff fast path and extension filtering
- Add the analysis pipeline after filtering
- Print each finding with format: `[SEVERITY] path:line — rule_id: message`
- Print summary: `N findings (X critical, Y errors, Z warnings)`
- Return `fmt.Errorf("commit blocked: %d critical/high findings", count)` if blocking findings exist
